# 项目主档案

## 项目摘要

- 项目名称：AIPD 能力建设与落地优化
- 项目背景：围绕 DJI 深度感知组的 AIPD 方向开展工作，目标是在能力、精度、效率和落地效果上持续提升整体竞争力。
- 当前阶段：入职初期的现状理解、方案熟悉与方向梳理阶段

## 这个项目为什么重要

- AIPD 是深度感知能力的重要组成部分，直接影响精度、效果和商用落地表现。
- 该方向不仅涉及模型效果，也涉及 benchmark、sim2real、训练增强、公版化和部署效率等系统性能力。
- 如果能把方案理解透，并找到关键改进点，能够同时提升交付效率和中长期竞争力。

## 当前目标

- 尽快理解 DJI 当前 AIPD 的整体方案、精度水平和实际落地情况。
- 梳理已有 benchmark、实验路径、数据增强方式、仿真方案和部署链路。
- 在充分理解现状后，逐步提出基于过往经验的判断、建议和优化方向。

## 当前范围

- 当前包含：
  - AIPD benchmark 的设计、理解与评估
  - sim2real 的 raw 域仿真，包括 noise 建模与 RGB 到 pdraw 的退化
  - 多个 sensor 下的模型泛化性分析与提升
  - 模型训练数据增强代码的公版化思路整理
  - 交付与部署人效提升
  - AIPD 精度、效果及其与后端策略融合方式的理解与优化
- 当前不包含：
  - 暂无明确排除项，后续会随着职责边界清晰而补充

## 当前主要交付

- 对现有 AIPD 方案的系统理解
- 对关键技术链路的调研结论和阶段性判断
- 可落地的实验方案、分析结果或优化建议
- 与交付直接相关的问题优先推进和闭环

## 成功标准

- 能够清晰说明 DJI 当前 AIPD 的方案、精度水平与落地方式。
- 能够识别当前链路中的关键问题、瓶颈和优先级。
- 每周至少产出一项初步进展，例如实验结果、调研结论或阶段性判断。
- 在交付优先的前提下，逐步沉淀对预研方向的有效判断。

## 风险

- 入职初期信息不完整，容易因为背景不足而判断失真。
- 相关链路较长，涉及仿真、训练、评估、部署与策略融合，理解成本较高。
- 如果优先级管理不清晰，容易在交付和预研之间分散精力。

## 待验证问题

- DJI 当前 AIPD 的核心方案、精度水平和商用表现分别是什么。
- 当前 benchmark 是否足以真实反映模型能力和落地效果。
- sim2real 的 raw 域仿真链路中，哪些环节最影响最终效果。
- 多 sensor 泛化、训练增强公版化和部署效率中，最值得优先推进的突破点是什么。
- AIPD 与后端策略的融合边界和关键约束是什么。

## 备注

- 当前仍处于快速熟悉业务、团队分工和历史方案的阶段，这个文件后续会随着理解加深持续更新。
